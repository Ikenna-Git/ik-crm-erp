# Landing Page Conversion Review

Date: 2026-06-07

## Current Landing Page Weaknesses Addressed
- Hero was broad, but not explicit enough about Civis being an AI-native CRM/ERP command centre.
- Trust messaging needed to stay premium without drifting into unproven compliance claims.
- Pricing needed stronger honesty around billing readiness.
- The page needed clearer problem → solution → governance → CTA flow.

## Improved Messaging
- Civis is positioned as a governed, AI-guided operating layer.
- CRM, finance, HR, operations, approvals, and admin boundaries are framed as one connected story.
- Copy is shorter, calmer, and more explicit about what is live versus setup-required.

## Hero Rationale
- Lead with category and user outcome.
- Avoid generic “all-in-one” language without governance context.
- Use “command centre”, “governed workspace”, and “AI-native” as differentiating cues.

## CTA Rationale
- Primary CTA: start workspace
- Secondary CTA: review pricing or trust
- Keeps the path clear without implying unsupported self-serve checkout

## Trust Section Rationale
- Show real trust signals:
  - founder-locked super-admin model
  - workspace-scoped admins
  - route guards and org isolation
  - honest rollout posture
- Avoid fake certifications or unsupported enterprise claims

## Conversion Risks
- If pricing feels too guarded, some buyers may want a stronger sales/contact path later.
- If preview-only modules are too visible, they can distract from live-ready flows.
- If provider-backed states are not validated live, demos can still fail despite better copy.

## Mobile Risks
- Dense premium sections can create vertical fatigue.
- Header and hero CTAs need continued live mobile review.
- Record dialogs and privacy-lock cards need browser validation on small screens.

## Claims Avoided Intentionally
- live public billing
- proven backup / restore
- certified compliance
- fake customer logos or testimonials
- fake AI autonomy claims

## Future A/B Test Ideas
- Hero headline: command centre vs operating layer
- CTA: start workspace vs book guided rollout
- Pricing CTA language
- Trust section ordering before vs after product modules
- AI-native section placement above vs below module story
